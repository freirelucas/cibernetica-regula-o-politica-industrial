# Sessão 2026-05-29 — Fase D · Author network completo

## TL;DR

Implementada Fase D do roteiro: rede completa de autoria sobre 880 works com authorships, top-30 cross-axis ranking, tag `obra de autor-ponte` no Rayyan, sondagem por autor (snowball) + tradições adjacentes (Polanyi/Schumpeter/Hirschman/complexity/governmentality), nova seção §10·6 "As pontes humanas". 30 testes verdes. Budget OpenAlex: 845/10000.

## Pipeline implementado

`src/author_network.py` (novo, 320 linhas):
1. **Mineração do cache** (`mine_cache_works`): recupera 880 works com authorships já no cache.
2. **Enriquecimento de títulos** (`enrich_works_titles`): batch fetch /works?filter=openalex:W1|W2|... com select rico (4 fetches novos; 879/880 com título).
3. **Classificação por eixos** (`classify_axes`): vocab match em título — VOCAB_CYB (organizacional + fundacional), VOCAB_REG (tools of government, regulation, governance), VOCAB_POLIND (industrial policy, mission-oriented, economic planning, evolutionary).
4. **Agregação por autor**: n_works_total, n_per_axis (C/R/P).
5. **Cross-axis score**: entropia normalizada da distribuição (1 = balanceado, 0 = concentrado).
6. **Eager fetch /authors/{id}**: para os 817 autores únicos — h_index, institutions, concepts, works_count, cited_by_count. 817 fetches novos.
7. **Grafo de coautoria + Leiden**: 743 autores conectados, 68 comunidades (Leiden via igraph+leidenalg).
8. **Output**: `data/author_network.json` (440 KB) com `top_cross_axis` (top-60) + `authors` (todos).

`src/build_site.py:inject_author_network_numbers`: substitui AUTHORNET_* tokens + renderiza tabela top-30 a partir do JSON. Future scaling = re-rodar pipeline, zero HTML manual.

`src/site_template.html` §10·6 nova:
- Lead: contraste pontes textuais (rejeitadas pelo null pairwise) × pontes humanas (autores que coexistem nos 3 silos).
- Tabela top-30 (oa link, score, distribuição C/R/P, h-index, community).
- Callouts: estritas vs soltas, sondagens dirigidas, caveats do classificador.

`src/build_rayyan.py:tag_author_bridge`:
- TOP_N_AUTHOR_BRIDGES=30, MIN_CROSS_AXIS_SCORE=0.35 env-overridable.
- Aplica tag `obra de autor-ponte` a obras escritas por autor no top-N.
- Rationale: "autor X — cross_axis_score=0.YY".
- 34 obras tagueadas (de 206 consolidadas).

`tests/test_author_network.py` (novo): 5 testes — chaves no JSON, top-30 score>0, OA ids válidos, tag aplicada no CSV, manifest com thresholds, snowball+adjacent files presentes.

## Caveats reconhecidos

1. **Classificador conservador**: só vocab em título. Autores cujos works tratam de Cyb/Reg/PolInd no abstract ou concepts (não no título) são sub-classificados. **Próxima sondagem natural**: estender para OpenAlex concepts ponderados (cada work tem concepts com score) — captaria contribuições temáticas sem keyword no título. Estimativa: +500-1500 fetches para enriquecer com concepts dos works que não estão completos no cache. Não feito nesta fase.

2. **Cross-axis score com poucos works classificados**: alguns autores no top têm 1-4 works classificados (não 50). O score é matematicamente correto mas sample-size-fraco. Honesto, mas merece um filtro adicional `min_classified_works ≥ N` para versão posterior.

3. **Coautoria sobre 743 nodes** (não todos os 817 — autores sem coautor não entram no grafo). 68 comunidades = muitos cluster pequenos + alguns grandes. Não exploramos visualmente.

4. **Snowball limitado a 50 obras/autor**: top-cited only. Para alguns autores prolíficos (Boulding 52 works, Margetts 51) isso já cobre boa parte; para autores muito ativos pode subestimar a expansão.
